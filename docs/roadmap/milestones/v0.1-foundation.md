## Goal

Stand up the core pipeline: ingest activity files, persist them, render a minimal dashboard. Prove the data model can support the analytics that come next.

## Scope

- [x] `.fit` file parser (`fit-file-parser`)
- [x] `.gpx` file parser (`fast-xml-parser`)
- [x] Activity upload endpoint + list/detail pages
- [x] SQLite persistence via Drizzle ORM
- [x] Base Next.js 16 (App Router) + Tailwind v4 + shadcn/ui scaffolding

## Technical notes

- SQLite via `better-sqlite3` chosen to keep the app deployable without external infra.
- Raw parsed payload is stored in `rawDataJson` so later features (map view, W′ balance) can derive from per-second data without re-parsing.
