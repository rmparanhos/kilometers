## Goal

Turn ingested activities into a real training-load signal. Implement the Banister impulse-response model end-to-end (load → CTL/ATL/TSB) with a sensible fallback chain so users without an HR profile still get useful numbers.

## Scope

- [x] Banister TRIMP load calculation (HR Max + HR Rest)
- [x] Linear hrTSS fallback (Avg HR + LTHR, Manzi et al. 2009)
- [x] Duration-only fallback (60 TSS/hour) when HR data is missing
- [x] Auto-selection of the best available model per activity
- [x] CTL / ATL / TSB performance manager chart
- [x] Training zone classification with contextual advice (peak / fresh / neutral / fatigued / overreached)
- [x] Garmin Connect sync — download raw `.fit` + recalculate separately
- [x] Activity calendar (training load heatmap)

## Technical notes

- Load and metrics are split: download stores raw files; recalculate re-derives load from the HR profile. This lets profile edits retroactively update history without refetching from Garmin.
- Model selection lives in `src/lib/training/metrics.ts` — priority order: Banister → hrTSS → duration.
