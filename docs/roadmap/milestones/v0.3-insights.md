## Goal

Layer performance-analytics features on top of the load baseline, document the science so the numbers aren't a black box, and open the app to multiple users.

## Scope

- [x] VO₂max estimation from submaximal HR (Swain et al. 1994) with 28-day EWMA trend
- [x] Critical Speed model (hyperbolic, Monod & Scherrer 1965) + race time predictions (5 km / 10 km / half / marathon)
- [x] Weekly volume chart with 8-week rolling average
- [x] Activity detail page — pace & HR chart, lap splits, weather snapshot
- [x] Multi-user support with session switching (`/admin`)
- [x] Training science documentation page (`/docs`) — models, assumptions, references

## Technical notes

- Critical Speed fit uses duration bins (not a Pareto front) — the bin approach proved more stable for runners with sparse max-effort data. See commit `eef28e8`.
- VO₂max uses a 28-day EWMA to smooth per-activity noise while staying responsive to real fitness changes.
- Multi-user is cookie-based; all dashboards and APIs resolve the active user through `getCurrentUser`.
