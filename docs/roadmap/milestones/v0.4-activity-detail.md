## Goal

Make the activity detail page the place where a runner understands *this specific run*: where they cracked, where they were strong relative to their own history, and what the route actually looked like.

## Scope

- [x] Km split comparison vs best run — per-km split bars with a reference line for the runner's best-ever split at that km index, delta annotations
- [x] Interactive route map — polyline from `rawDataJson` GPS coordinates, color-coded by pace quartile along the trace

## Technical notes

- **Km split comparison** ships on branch `feature/km-split-compare` (commit `e1ffc2f`). Splits are derived in [src/lib/training/km-splits.ts](../../../src/lib/training/km-splits.ts) from per-second records; the chart uses a Recharts `ComposedChart` with `Bar` + `ReferenceLine`. Best-split lookup aggregates across the user's activity history, bucketed by km index.
- **Map view** ships on branch `feature/map-view`. Uses Leaflet + `react-leaflet` with CartoDB Positron (`light_nolabels`) tiles for a minimalist background that highlights the trace. `NormalizedRecord` was extended with optional `lat` / `lon`; `.fit` positions are auto-converted semicircles→degrees by `fit-file-parser` (no manual math). Polyline is split into segments and colored by pace quartile computed from this activity's own distribution (adaptive, not absolute thresholds). Segments downsampled to 2000 points for long trails. Next.js 16 Server Components can't use `dynamic({ ssr: false })` directly, so a thin client wrapper [RouteMapClient.tsx](../../../src/components/charts/RouteMapClient.tsx) defers the import.
