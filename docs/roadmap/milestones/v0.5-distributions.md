## Goal

Show *how the runner's training is distributed* — by distance and by cadence — so patterns (bias toward short runs, drifting away from 180 spm) become visible at a glance.

## Scope

- [ ] Distance distribution histogram (dashboard) — activity count per runner-friendly distance bucket, controlled by a shared time-window selector
- [ ] Cadence distribution histogram (dashboard) — aggregate of `avgCadenceRpm` across activities, 2-spm bins, 180 spm reference line
- [ ] Per-activity cadence histogram (activity detail) — distribution of instantaneous cadence within the run, 2-spm bins, 180 spm reference line
- [ ] Dashboard time-window selector (`30d` / `90d` / `YTD` / `all`) via `searchParams`, shared across all dashboard charts that need it

## Technical notes

- Data is already persisted: `activities.distanceM` and `activities.avgCadenceRpm` for the aggregates; `NormalizedRecord.cadenceRpm` (from `rawDataJson`) for the per-activity chart.
- No schema change.
- Buckets are opinionated (not evenly spaced) — `<3, 3-5, 5-8, 8-12, 12-18, 18-25, 25+` km — reflecting real running volume distribution.
- Cadence bins are 2 spm wide; range derived from actual data (min→max rounded to even).
- Many `.gpx` exports don't include cadence. Aggregate cadence chart hides when fewer than 2 activities have valid cadence; per-activity chart hides when ≥ 60% of records miss cadence.
- Window selector is server-side (searchParams) to stay consistent with the rest of the app's SSR pattern and make state URL-shareable.
